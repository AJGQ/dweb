Abstract:

Para informação mais volumosa ou privada é inademissível usar o método GET para envio da mesma, pois essa informação tem um limite de caracters e não há proteção nenhuma. Para circunstâncias desse género usa-se o método POST.

Neste trabalho foi feito um single-page website para uma aplicação de gestão de tarefas, onde é possível registar uma tarefa e visualizar as já registadas previamente.

Deliverables

  • erro.pug - mensagem de erro
  • index.pug - pagina principal do website
  • layout.pug - complementar para outras paginas
  • pr.dtd - descreve o pr
  • pr.xsl - stylesheet para visualizar em html
  • stylesheets - diretoria de todas as stylesheets
  • stylesheets/w3.css - stylesheet do w3schools
  • tarefas.js - servidor para o website
  • tarefas.json - base de dados para as tarefas
